# Core Loop

## Loop overview

### 1. Goal Intake
Capture what the user wants in plain language.

### 2. Domain Framing
Interpret the goal as a validation-heavy research problem.

### 3. Research Spec Compilation
Define:
- objective
- assumptions
- constraints
- evidence needs
- risk boundaries
- validation needs

### 4. Candidate Generation
Generate multiple directions:
- baseline
- conservative
- exploratory
- hybrid if appropriate

### 5. Evidence Planning
Define:
- what data is required
- what is optional
- what evidence gaps exist
- what cannot be skipped

### 6. Validation Plan Generation
Define:
- tests
- metrics
- windows
- failure conditions
- comparison targets

### 7. Audit / Rejection
Critique candidates for:
- weak assumptions
- leakage risk
- unrealistic validation
- over-complexity
- hidden dependencies
- false confidence

### 8. Recommendation
Return:
- best current candidate
- runner-up
- rejected candidates
- conditions
- unknowns
- next validation steps

### 9. Re-evaluation Trigger
Define what new evidence would change the recommendation.

## Core product truth
The product is not “chat → answer.”
The real product is this loop.
