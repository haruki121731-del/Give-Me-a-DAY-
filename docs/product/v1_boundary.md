# v1 Boundary

## v1 in scope
- investment research / strategy validation / hypothesis-testing system design
- user goal intake
- domain framing
- research spec compilation
- evidence and data requirement planning
- candidate generation
- validation plan generation
- audit / rejection output
- recommendation report generation
- explicit assumptions, conditions, and next validation steps

## v1 out of scope
- generic business automation
- broad “build anything” workflows
- live auto-trading execution by default
- universal deployment infrastructure
- one-click production orchestration
- broad no-code workflow builders
- fake certainty around performance
- pretending paper validation equals production truth

## Expected v1 output
A recommendation package containing:
- user goal summary
- research spec
- candidate set
- evidence requirements
- validation plans
- audit findings
- rejected options
- best current candidate
- runner-up
- recommendation conditions
- next validation steps

## v1 principle
v1 should be narrow, deep, and honest.
If a feature makes the product broader but weaker, it should stay out.
